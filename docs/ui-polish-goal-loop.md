# UIブラッシュアップ・ゴールループ

対象: 戦略マップ画面、軍制成長画面

基準文書: [世界観ガイド](worldview-guide.md)

## 目的

戦略画面と成長画面を、「スライム育成ゲーム」ではなく「人間国家が弾性軍制を運用する戦争シミュレーション」として見える状態へ寄せる。

このループで磨くものは、単なる装飾ではない。

- プレイヤーが国家の司令部にいると感じること
- 内政、外交、補給、軍制が次の戦闘条件へつながること
- 成長候補が人間の制度、訓練、教範として読めること
- 画面上の生成画像、アイコン、文言が人間国家の軍事語彙に揃うこと
- スマホ画面で読め、押せ、はみ出さないこと

## ゴールモードの進め方

1周を小さく回す。1周で触るのは原則として1テーマだけにする。

例:

- 成長カードの文言とレイアウトだけ
- 戦略マップの地域ノードと資源表示だけ
- 戦略アクションのアイコンとラベルだけ
- 成長画面の背景、額縁、軍制スロットだけ

大きく作り替えるより、毎周「違和感を1つ消す」ことを優先する。

## 1周の手順

### 1. 現状取得

ローカルまたは公開版で、対象画面を実際に開く。

必ず確認するサイズ:

- `375x667`: iPhone SE相当
- `390x844`: 標準的なiPhone相当
- `1280x720`: 横長PC相当

使うコマンド:

```bash
npm.cmd run smoke:visual
```

必要なら公開版も確認する。

```powershell
$env:SMOKE_URL='https://yohei2000.github.io/slime-battle-sim/'
npm.cmd run smoke:visual
Remove-Item Env:SMOKE_URL
```

成果物:

- `artifacts/visual-smoke/*.png`
- `artifacts/visual-smoke/summary.json`

### 2. 世界観監査

対象画面を見て、次の表で採点する。

| 観点 | 0点 | 1点 | 2点 |
| --- | --- | --- | --- |
| 国家感 | 魔物・生物・抽象UIに見える | 軍事っぽいが国家の気配が弱い | 国家の司令部、軍制、補給、外交が見える |
| 人間ベース | スキルや資源が生体器官に見える | 一部だけ人間制度に見える | 全体が制度・訓練・兵站として読める |
| 戦闘接続 | 何が戦闘へ効くか分からない | 効果はあるが手触りが弱い | 形、密度、結束、応力、命令遅延へ明確につながる |
| 画面階層 | どこを見るべきか迷う | 主情報は分かるが混んでいる | 次に押すもの、読むもの、結果が明確 |
| 可読性 | スマホで潰れる・切れる | 読めるが窮屈 | スマホでも主要情報が読める |
| 画像品質 | 装飾に見える、または生成画像が活きていない | 一部は効いている | 背景・アイコン・カードが世界観を強化している |
| 旧スライム漏れ | スライム、胞子、粘液、触手などが表に出る | 一部に残る | 内部比喩に閉じ、表は軍事語彙になっている |

各画面で合計14点満点を目安にする。

次の状態なら次の実装修正へ進む。

- どれか1項目が0点
- 合計が10点未満
- スマホで主要UIが切れている
- 旧スライム語彙が画面上に出ている
- 画像があるのに世界観を支えていない

### 3. 修正テーマを1つ選ぶ

修正テーマは、次の型に落とす。

```text
今回の違和感:
対象画面:
原因:
直す範囲:
触らない範囲:
完了条件:
```

例:

```text
今回の違和感:
成長候補カードが制度ではなく魔法スキルに見える。

対象画面:
軍制成長画面

原因:
カード名、アイコン、説明文の主語が人間組織ではなく効果名になっている。

直す範囲:
src/growth/slimeSkills.ts の表示名と説明文
src/game/SlimeGrowthScene.ts のカード表示密度

触らない範囲:
戦闘シミュレーション値、内部型名

完了条件:
3候補すべてが軍学校、工兵、兵站、参謀などの制度として読める。
390x844でカード3枚が切れず、主要文が読める。
```

### 4. 実装

実装時の優先順位:

1. 表示文言と情報設計
2. 画像・アイコン・背景
3. レイアウトとスマホ可読性
4. アニメーションと質感
5. 内部データ名の整理

内部名の `Slime` 系は、広範囲リネームのリスクが大きいので原則として後回しにする。
まずはユーザーに見える表示を整える。

### 5. 検証

最低限:

```bash
npm.cmd run build
npm.cmd run smoke:visual
```

目視確認:

- `artifacts/visual-smoke/growth-iphone-se.png`
- `artifacts/visual-smoke/growth-iphone-13.png`
- `artifacts/visual-smoke/strategy-iphone-13.png`
- `artifacts/visual-smoke/growth-desktop.png`

確認すること:

- canvasが1枚ある
- console errorがない
- `failures` が空
- 主要情報が画面下で切れていない
- ボタン文言が読める
- 生成画像が背景やアイコンとして見えている
- 古いスライム語彙が画面上に出ていない

### 6. ループ終了判定

各画面について、次を満たしたら一旦終了する。

#### 戦略マップ

- 地域ノードが「地形と国家戦略の場所」に見える
- 内政アクションが軍制・兵站・訓練として読める
- 外交アクションが通行、補給、威圧、信用として読める
- 選択地域、資源、次の戦闘プレビューの関係が分かる
- iPhone SE相当で下端が切れない

#### 軍制成長画面

- タイトル、カード、スロットが「軍制成長」に見える
- 候補スキルが人間の制度・訓練・組織として読める
- 効果が軍塊の形、密度、結束、応力、命令遅延へつながる
- 3候補がスマホで読める
- 画像が生体器官ではなく軍事・制度モチーフに見える

#### 共通

- `npm.cmd run build` 成功
- `npm.cmd run smoke:visual` 成功
- 公開版へ出す場合は GitHub Pages の実画面 smoke も成功
- 世界観監査で各画面12点以上
- 0点項目がない

## 重点バックログ

優先度A:

- 成長画面のカード文言をすべて「人間の制度・部隊・教範」へ寄せる
- 戦略マップの資源名を `糧秣 / 予備兵 / 工兵資材 / 装甲材 / 参謀知` に寄せる
- 戦略アクションのラベルを文字化け・旧語彙なしで統一する
- スマホで押せるボタンサイズと読みやすい文字量にする

優先度B:

- 戦略ノード画像を「地形 + 軍事拠点」の絵として強める
- 成長スロットを「軍制スロット」よりも「教範・組織枠」に見えるよう調整する
- 画面遷移ボタンを「戦略へ」「戦闘へ」より、司令部らしい短い語へ整える
- 戦闘プレビューに、地形・外交・内政がどう効くかを1行で出す

優先度C:

- 内部型名の `Slime` 系を表示層から完全に隔離する
- READMEや古い設計書の旧語彙を段階的に更新する
- スクリーンショット比較の自動化を強化する

## ゴールモード開始用プロンプト

次の指示で開始する。

```text
docs/worldview-guide.md と docs/ui-polish-goal-loop.md を基準に、戦略マップ画面と軍制成長画面のUIブラッシュアップをゴールモードで進めてください。

1周ごとに、現状スクリーンショット確認、世界観監査、1テーマの修正、build、smoke:visual、目視確認を行ってください。

優先順位は、スマホ可読性、旧スライム語彙の排除、人間国家の軍事語彙への統一、生成画像の世界観反映、戦闘への効果接続です。

各画面の世界観監査で12点以上、0点項目なし、npm.cmd run build成功、npm.cmd run smoke:visual成功になるまでループしてください。
```

## 各ループの報告フォーマット

```text
ループ:
対象:
見つけた違和感:
修正:
確認:
残り:
次の一手:
```

報告は短くてよい。重要なのは、次の修正に進める証拠を残すこと。
