# Slime Battle Sim

公開版: https://yohei2000.github.io/slime-battle-sim/

設計資料:

- [スライム挙動仕様書](docs/slime-behavior-spec.md)

Slime Battle Sim は、個別ユニットを操作するRTSではなく、軍勢そのものをスライム状の連続体として操作する戦術シミュレーションです。

ピンチアウトで軍勢を展開し、敵を包囲します。展開すると支配面積が広がりますが、戦線が薄くなり突破されやすくなります。

ピンチインで軍勢を密集させ、突破力を高めます。密集すると局所圧力は上がりますが、過密・疲労・包囲リスクが増えます。

勝敗は、広がるか、固まるか、どこで接触し、どこを包み、どこを破るかによって決まります。

## 起動

```bash
npm install
npm run dev
```

本番ビルド:

```bash
npm run build
npm run preview
```

Viteのbaseは相対パス指定なので、`dist/` をGitHub Pagesへそのまま配置できます。

## スマホ操作

- タップ: 自軍スライムを選択
- 1本指ドラッグ: 軍勢全体の進行方向と移動先を指示
- ピンチアウト: 横へ展開。ZOCと包囲力を増やす
- ピンチイン: 密集。局所密度と突破力を増やす
- ピンチアウトしながら前進: 幅を広げながら包囲前進
- ピンチインしながら前進: 密度と正面圧力を集中する突破
- 2本指を回す: 軍勢の幅を保ちながら戦線を回転
- 片方の指だけ前へ出す: 指を置いた側の翼だけを前進
- 選択中スライム以外の空白をドラッグ: カメラパン
- 未選択時のピンチ: カメラズーム
- 左下の `+` / `−` / `全`: カメラ拡大、縮小、全体表示
- 左下の `応力`: 危険リンクのノード番号、負荷率、耐久度、主因を表示
- 上部の `Ⅱ` / `▶`: 一時停止と再開
- 上部の `×1`: `×1 → ×1.5 → ×0.5` の速度変更

形状命令には、疲労、圧力、交戦状態、結束低下、形状変化量に応じた伝達遅延があります。

二本指ジェスチャーは、ピンチ量、二指の重心移動、回転角、左右それぞれの前進量を同時に解析します。個別ユニットへ命令するのではなく、ArmySlimeの幅、密度、向き、左右翼の前進差、接触面を変更します。

## ゲームの設計思想

ゲームの中核は、次の矛盾です。

- 広がると包囲しやすいが、局所密度が落ちて中央を破られやすい
- 固まると突破しやすいが、接触正面が頭打ちになり、側面を包まれやすい
- 包囲は時間とともに士気・疲労・命令速度へ効くが、薄い包囲線は一点突破される
- 突破は短時間で形勢を変えられるが、失敗すると密集状態のまま疲労・士気・結束を失う

HUDは密度、幅、疲労、圧力、`gapRisk`、`crowding`、包囲力、突破力を常時表示します。危険が高まると「戦線の隙間」「過密継続」「包囲線が薄い」などを警告します。

選択中スライムのリンクは常時表示されます。青緑は低負荷、黄は注意、橙は高負荷、点滅する赤は現在の靱性を超えた状態です。線の欠けは累積損傷、短い矢印は敵圧の方向、点線は予測分裂面を示します。HUDには最も危険なノード間と、「接触圧・展開・疲労・低結束・低士気」のうち現在の主因を表示します。

戦場は`1000 × 650`の接触重視サイズです。初期配置も中央寄りにし、長い行軍より接触後の変形・押し合い・包囲・破断へ時間を使う構成にしています。

各スライムの頭上には兵力と士気を数値表示します。兵力はArmySlimeの`mass`で、分裂時には子スライムへ保存・分配されます。士気が25以下になると、そのスライムは敗走してプレイヤー命令を受け付けず、敵から離れる方向へ移動します。敗走中は戦闘力とZOCが大幅に低下します。

## シミュレーション

各軍勢は1つの `ArmySlime` です。MVPでは自軍1体・敵軍1体ですが、シミュレーションAPIは複数のArmySlimeへ拡張できるよう責務を分離しています。

- `SlimeNode`: 外周20点、中間14点、内層8点、中心1点からなる制御点
- `SlimeLink`: 同一層の隣接点と隣の層の近傍だけを結ぶ局所メッシュばね
- `SlimeParticle`: 内部の兵士表現。選択や個別命令は不可
- `ContactPatch`: 点ではなく面として接触正面、圧力、密度、frontageを保持
- `SlimeOrder`: 命令伝達遅延と実行状態を保持

毎フレーム、形状目標、流れ、ばね、分離、敵接触圧、地形境界を合成してノードを積分します。内部粒子は周辺ノードへ追従し、軍勢全体がばらばらの部隊ではなく連続した塊として動きます。

リンクは中心へ放射状に直結しません。外周・中間層・内層を網目状に結び、敵圧を受けた場所の近傍だけへ応力が伝わります。そのため正面突破では正面メッシュ、片翼への圧力ではその翼のメッシュが先に黄・橙・赤へ変化します。

敵ZOCは侵入できない境界です。輪郭ノードがZOCへ触れると、法線方向の移動は停止し、接線方向の成分だけが残ります。これにより軍勢は相手をすり抜けず、接触正面が潰れ、側面ノードが相手の輪郭に沿って流れます。展開・包囲姿勢は接線方向へ回り込みやすく、密集・突破姿勢は正面へ圧力を蓄積します。

スライムはリンクばねによって一体性を保ちます。幅を広げただけでは破断しません。敵との接触面やZOCから局所圧力を受け、リンク応力が現在の実効靱性を超え続けると、その場所の耐久度が低下します。近接した複数リンクの亀裂が連結した時点で、亀裂位置と圧力方向に沿って2個のArmySlimeへ分裂します。展開状態は密度と結束が低いため敵圧に弱くなりますが、分裂の直接原因は幅ではなく局所応力です。

戦闘力は総質量だけでは決まりません。

```text
combatPower =
  activeFrontage
  × localDensity
  × morale
  × cohesion
  × posture
  × fatigue
```

接触面に出られない後方質量は15%だけ戦闘力へ加算されます。そのため、密集は突破圧力には強い一方、長時間の全面戦闘では効率が頭打ちになります。

## 敵AI

敵も個別兵士ではなくArmySlime単位で判断します。

- 相手の`gapRisk`が高い: 中央の弱点へ突破
- 相手が過密で側面へ回れる: 展開・包囲
- 自分が包囲された: 一点突破
- 士気または結束が崩れた: 後退
- 疲労が高い: 保持

AIの姿勢変更もプレイヤーと同じ命令遅延、形状変形、疲労、結束、ZOC、接触面ルールに従います。
